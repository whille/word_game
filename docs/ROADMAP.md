## 技术债务

| 区域 | 问题 | 优先级 |
|------|------|--------|
| 测试 | 缺少引擎单元测试 | 高 |
| 测试 | 编辑器无 e2e 覆盖 | 中 |
| 代码 | Editor panel 组件重复的 form 组件可提取 | 低 |
| 代码 | GameShell.tsx 接近 400 行，需拆分 | 中 |
| 数据 | 三个关卡共享背景图 key，新关卡需新图 | 中 |
| 性能 | 编辑器 + 游戏代码全量打包，可懒加载 | 低 |

## 架构原则

1. **纯静态** — 无后端，localStorage + URL 参数 = 全部存储
2. **JSON 驱动** — 关卡 = JSON 文件，编辑器 = JSON 编辑器
3. **无环依赖** — `data → engine → store → ui`，store 绝不 import ui
4. **Web 优先** — 移动端响应式，PWA 离线可用
5. **开源免费** — 不设付费墙，社区贡献驱动
